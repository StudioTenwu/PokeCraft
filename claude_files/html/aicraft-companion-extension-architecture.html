  • Independence: Separate from deployment system

High-Level Flow

  1. Export Phase: Backend generates extension package with agent data
  2. Distribution Phase: Parent downloads .zip file containing extension
  3. Installation Phase: Child loads unpacked extension in Chrome
  4. Usage Phase: Child opens side panel and chats with agent
üéØ Architecture Principles & Constraints ‚ñ∂
  • Side Panel is Mandatory: Must use Chrome Side Panel API (chrome.sidePanel)
  • Simplicity First: No complex features - just avatar, personality, chat
  • Offline-Capable: Agent data stored locally, LLM calls are only external dependency
  • Manifest V3: Use modern Chrome extension format
  • No Backend Dependency: Extension operates independently after export
  • Pokemon Aesthetic: Game Boy Color retro theme from AICraft
  • Child-Friendly: Simple, intuitive UI suitable for children

LLM Integration Strategy

Option 1 (Recommended): Claude Agent SDK

  • ‚úÖ Matches AICraft backend pattern
  • ‚úÖ No API key management needed (uses Claude Code credentials)
  • ‚ùå Requires Claude Code to be installed on child's computer

Option 2: Direct Anthropic API

  • ‚úÖ Works independently of Claude Code
  • ‚ùå Requires API key (security concern)
  • ‚ùå Parent must fund API usage

Option 3: Backend Proxy

  • ‚úÖ Secure API key management
  • ‚úÖ Parent controls costs
  • ‚ùå Requires running AICraft backend
  • ‚ùå Not "standalone" anymore

Recommendation: Start with Option 3 (backend proxy) for security and simplicity, then evaluate Option 1 if Claude Code adoption is feasible.

Message Flow States

  1. Idle: Input enabled, waiting for user
  2. Sending: Input disabled, message appended to UI
  3. Waiting: Loading indicator, calling LLM
  4. Streaming: Response chunks appearing
  5. Complete: Full response shown, input re-enabled
  6. Error: Error message shown, retry button

TDD Principles for Extension

  • Test Behavior, Not Implementation: Test what the extension does, not how
  • Mock Chrome APIs: Use chrome-mock library for testing
  • Isolate Components: Unit test each module independently
  • E2E Tests: Use Puppeteer for side panel interaction tests

describe('Extension Integration', () => {

  beforeEach(() => {
    chrome = createChromeMock();

  test('full chat flow', async () => {
    // 1. Install extension
    await triggerInstallEvent(chrome);

End-to-End Tests ‚ñ∂

E2E Test Tools

Test Flow

describe('E2E: Extension Usage', () => {
  let browser, page;

  beforeAll(async () => {
    browser = await puppeteer.launch({
      headless: false,
      args: [
        '--load-extension=./dist/aicraft-pokemon-oak',
        '--disable-extensions-except=./dist/aicraft-pokemon-oak'
      ]
    });
    page = await browser.newPage();
  });

  test('child can chat with agent', async () => {
    // Open side panel
    await page.evaluate(() => {
      chrome.sidePanel.open();
    });

    // Wait for panel to load
    const panelPage = await browser.waitForTarget(
      target => target.url().includes('panel.html')
    );

    // Type message
    await panelPage.type('#message-input', 'Tell me about Pokemon');
    await panelPage.click('#send-button');

    // Wait for response
    await panelPage.waitForSelector('.message.assistant', {
      timeout: 10000
    });

    const responseText = await panelPage.$eval(
      '.message.assistant',
      el => el.textContent
    );

    expect(responseText).toContain('Pokemon');
  });
});

Side Panel Opening Patterns

// Option 1: User clicks toolbar icon
chrome.action.onClicked.addListener(() => {
  chrome.sidePanel.open();
});

// Option 2: Automatic on install
chrome.runtime.onInstalled.addListener(() => {
  chrome.sidePanel.setPanelBehavior({
    openPanelOnActionClick: true
  });
});

// Option 3: Programmatic from content script
chrome.runtime.sendMessage({ action: 'openPanel' });

Storage Best Practices

// ‚úÖ GOOD: Batch updates
const updates = {
  chat_history: [...existingHistory, newMessage],
  last_interaction: Date.now()
};
await chrome.storage.local.set(updates);

// ‚ùå BAD: Multiple separate writes
await chrome.storage.local.set({ chat_history: [...] });
await chrome.storage.local.set({ last_interaction: Date.now() });

üîí Security & Privacy Considerations ‚ñ∂
  • No Telemetry: Extension does not send usage data back to AICraft servers
  • Local Storage: All agent data and chat history stored locally (chrome.storage.local)
  • API Key Security: If using backend proxy, no API keys in extension code
  • Content Security Policy: Strict CSP in manifest to prevent XSS
  • Minimal Permissions: Only request sidePanel and storage
  • No Content Scripts: Extension does not inject code into web pages
  • Child Safety: Chat responses filtered for age-appropriate content (LLM system prompt)

Recommended CSP

"content_security_policy": {
  "extension_pages": "script-src 'self'; object-src 'self'"
}
