üéØ What is Server-Sent Events (SSE)?

  • Protocol: HTTP-based unidirectional streaming from server to client
  • Transport: Text-based events over persistent HTTP connection
  • Browser API: Native EventSource interface
  • Reconnection: Automatic retry on connection failure
  • Method: GET only (EventSource limitation)
  • Format: event: name\ndata: json\n\n
  • Use Case: Real-time progress updates, live feeds
  • Advantage: Simple, standards-based, auto-reconnects

üíª Frontend (React + EventSource) ‚ñ∂

api.js - createAgentStream()

const url = `${API_BASE}/api/agents/create/stream?description=${encodeURIComponent(description)}`;

// EventSource only supports GET
const eventSource = new EventSource(url);

// Register event listeners
eventSource.addEventListener('progress', (event) => {
  const data = JSON.parse(event.data);
  callbacks.onProgress?.(data);
});

eventSource.addEventListener('complete', (event) => {
  const data = JSON.parse(event.data);
  callbacks.onComplete?.(data.agent);
  eventSource.close();
});

eventSource.addEventListener('error', (event) => {
  callbacks.onError?.(new Error('Stream failed'));
  eventSource.close();
});

Key Points

  • Native API: No external libraries needed
  • GET only: EventSource doesn't support POST
  • Auto-parse: event.data is string, must JSON.parse
  • Close manually: Call eventSource.close() when done
üîß Backend (FastAPI + AsyncGenerator) ‚ñ∂

main.py - Endpoint

@app.get("/api/agents/create/stream")
async def create_agent_stream(description: str, req: Request):
    async def event_generator():
        try:
            async for event in req.app.state.agent_service.create_agent_stream(description):
                event_name = event.get("event", "message")
                event_data = event.get("data", {})

                # SSE format
                sse_message = f"event: {event_name}\ndata: {json.dumps(event_data)}\n\n"
                yield sse_message

                await asyncio.sleep(0.01)  # Small delay
        except Exception as e:
            yield f"event: error\ndata: {json.dumps({'message': str(e)})}\n\n"

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "Connection": "keep-alive"
        }
    )

Key Points

  • GET method: Changed from POST to support EventSource
  • Query params: Data passed via ?description=...
  • Async generator: async def event_generator() yields events
  • SSE format: event: name\ndata: json\n\n (double newline!)
  • StreamingResponse: media_type="text/event-stream"

async def create_agent_stream(self, description: str):
    """Create agent with streaming progress updates."""
    try:
        # Step 1: Start
        yield {
            "event": "progress",
            "data": {"status": "started", "message": "Hatching your companion..."}

        }

        # Step 4: Save
        yield {
            "event": "progress",
            "data": {"status": "saving", "message": "Saving to Pok√©dex..."}
        }

        # ... database save ...

        # Step 5: Complete
        yield {
            "event": "complete",
            "data": {"agent": {...}}
        }

    except Exception as e:

            "event": "error",
            "data": {"message": str(e)}

Key Points

üìù SSE Message Format ‚ñ∂

Standard SSE Format

event: progress
data: {"status": "generating", "message": "Consulting with Claude..."}

event: complete
data: {"agent": {"id": "123", "name": "Pixel Pal", ...}}
Field Purpose Required
event: Event name for addEventListener No (defaults to "message")
data: Event payload (usually JSON) Yes
id: Event ID for resumption No
retry: Reconnection delay (ms) No
\n\n Event terminator (double newline) Yes

üîë Why We Migrated from POST to GET

‚ùå Old Approach (POST)

  • Method: POST with JSON body
  • API: Custom fetch with ReadableStream
  • Complexity: Manual stream parsing
  • Reconnection: Manual retry logic
  • Code: ~100 lines of stream handling

‚úÖ New Approach (GET + EventSource)

  • Method: GET with query parameters
  • API: Native EventSource
  • Complexity: Browser handles everything
  • Reconnection: Automatic retry built-in
  • Code: ~20 lines, cleaner
üêõ Common Issues & Solutions ‚ñ∂
Issue Cause Solution
405 Method Not Allowed Endpoint is POST but EventSource uses GET Change endpoint to @app.get(...)
Connection immediately closes Missing text/event-stream media type Use StreamingResponse with correct media_type
Events not received Missing double newline \n\n Format: event: name\ndata: json\n\n
Browser auto-reconnects endlessly Didn't call eventSource.close() Close on complete/error events
JSON parsing errors event.data is string, not object Use JSON.parse(event.data)

üí° Key Takeaways

  • SSE = Simple: Standards-based streaming with native browser support
  • GET Only: EventSource limitation requires query parameters
  • Async Generators: Perfect for streaming progress updates
  • Format Matters: event: name\ndata: json\n\n (double newline!)
  • Auto-Reconnect: Browser handles connection failures automatically
  • Close Explicitly: Call eventSource.close() when done
