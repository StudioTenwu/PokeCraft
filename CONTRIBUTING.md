# Contributing to Agent Engineering Playground

Thank you for your interest in contributing! This is an open-source educational project, and we welcome contributions of all kinds.

## Ways to Contribute

### 1. Add New Levels and Challenges
- Design new learning modules
- Create progressive difficulty challenges
- Write educational content

### 2. Improve the Environment Engine
- Add new entity types
- Implement physics features
- Optimize grid rendering

### 3. Design Agent Architectures
- Implement new reasoning patterns
- Add support for different LLM models
- Create tool libraries

### 4. Enhance UI/UX
- Improve visualizations
- Add animations
- Create better feedback mechanisms

### 5. Create Export Targets
- Build integrations with other environments
- Design export formats
- Document deployment processes

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/AICraft.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages: `git commit -m "Add: description of your changes"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Guidelines

### Code Style
- **Python**: Follow PEP 8, use `black` for formatting
- **JavaScript/React**: Use ESLint configuration provided
- Write clear, descriptive variable names
- Add comments for complex logic

### Testing
- Write tests for new features
- Ensure existing tests pass: `pytest` (backend) and `npm test` (frontend)
- Test in multiple browsers if changing UI

### Documentation
- Update README.md if adding features
- Document new APIs and components
- Add docstrings to Python functions
- Include usage examples

### Commit Messages
Use conventional commits format:
- `feat:` - New features
- `fix:` - Bug fixes
- `docs:` - Documentation changes
- `style:` - Code style changes
- `refactor:` - Code refactoring
- `test:` - Test additions/changes
- `chore:` - Maintenance tasks
